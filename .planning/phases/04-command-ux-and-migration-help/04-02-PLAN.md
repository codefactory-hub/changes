<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI flags -> init selection | Untrusted operator input chooses repo/global scope, layout style, optional home path, and current version |
| init orchestration -> filesystem | Init writes config, manifests, state directories, and repo `.gitignore` entries |
| README docs -> operator actions | Incorrect docs could steer operators toward wrong paths or conflicting layout choices |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | T | `internal/config/init_defaults.go` | mitigate | Keep strict validation that `--home` is only valid with `home`, and keep repo-home paths confined within the repo root so repo init cannot escape into arbitrary filesystem targets. |
| T-04-06 | T | `internal/app/init.go` | mitigate | Preserve authority checks and single-target write behavior while extending init results; only one repo-state ignore entry may be authoritative after init. |
| T-04-07 | R | `internal/cli/app.go` | mitigate | Print selected layout plus resolved config/data/state paths so operators can verify what was initialized without inferring hidden defaults. |
| T-04-08 | I | `README.md` | mitigate | Keep documentation synchronized with implemented help text and precedence rules so operators are not instructed to create unsupported layouts or conflicting override combinations. |
</threat_model>
