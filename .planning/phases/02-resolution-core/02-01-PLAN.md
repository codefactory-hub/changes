## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem and env -> resolver | Untrusted manifests, environment variables, and discovered directories cross into resolution logic |
| repo-local manifest -> resolved paths | Symbolic layout values become concrete filesystem paths that later commands may use |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | T | `internal/config/manifest.go` | mitigate | Reject unsupported keys, mismatched `scope` or `style`, and non-approved symbolic references during manifest decode. |
| T-02-02 | T | `internal/config/manifest.go` | mitigate | Normalize repo-local paths with `filepath.Clean`, reject non-local descendants, and canonicalize trusted anchors before candidate comparison. |
| T-02-03 | D | `internal/config/resolution.go` | mitigate | Classify malformed or legacy candidates into structured statuses so callers can stop safely without silent fallback behavior. |
| T-02-04 | R | `internal/config/resolution_types.go` | mitigate | Preserve per-candidate evidence and symbolic plus resolved manifest fields so later doctor and ambiguity flows can explain exactly what was detected. |
</threat_model>
