# Milestone v0.1.0-rc.2: Legacy Layout Repair

**Status:** ✅ SHIPPED 2026-04-07
**Phases:** 6-6
**Total Plans:** 1

## Overview

This milestone closed the most immediate operator gap left by `0.1.0-rc.1`: existing legacy repositories no longer require manual `layout.toml` authoring to become operational again. `changes` now exposes an explicit repo-local repair flow through `changes doctor --scope repo --repair`, preserves the single-authoritative-layout model, and keeps repair clearly separate from migration guidance.

## Milestone Summary

**Key Accomplishments**

- Added `changes doctor --scope repo --repair` so legacy repo-local layouts can be recovered without hand-authoring `layout.toml`.
- Reused one shared repo manifest writer for both init and repair so authoritative manifests are stamped with the same symbolic contract.
- Kept repair narrow, repo-local, and fail-loud on ambiguity while preserving the authoritative state ignore rule and avoiding migration side effects.
- Verified that repaired repositories immediately return to ordinary operation and documented when repair is appropriate versus when migration prompts are still required.

**Key Decisions**

- Repair remains under `doctor` rather than introducing a separate command family.
- Repair is allowed only when exactly one legacy repo-local candidate is repairable and no manifest-backed authority already exists.
- Repair mutates metadata only; it does not migrate data or dual-write across layouts.

**Issues Resolved**

- Existing legacy repositories no longer require manual manifest creation to recover from the manifest-backed operational boundary.
- The operator path from `legacy_only` diagnostics to a repaired authoritative repo-local layout is now explicit and test-covered.

**Issues Deferred**

- Validate user-completed migrations against expected source and destination layouts.
- Support future directory schema revisions beyond the first flexible-layout rollout.
- Add global-layout repair automation if operator demand justifies it.

**Technical Debt Incurred**

- Repo-local legacy repair is shipped, but broader migration validation and future schema evolution still need follow-on milestone work.
